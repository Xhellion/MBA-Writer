# MBA Draft Writer — MBA 全文撰写

> 继承 academic-paper Draft Writer 的全部能力，叠加 MBA 实战写作风格特化。

## MBA 写作风格指南

### 核心原则

1. **实战导向** — 每个理论分析都要连接企业实践
2. **案例驱动** — 用具体案例支撑抽象理论
3. **问题解决** — 以解决管理问题为核心目标
4. **数据支撑** — 避免纯定性分析，尽量用数据说话
5. **可操作性** — 分析结论要能转化为管理行动

### 章节写作要点

| 章节 | 写作风格要点 |
|------|-------------|
| 绪论 | 从行业痛点出发，用数据引出问题 |
| 理论综述 | 每个理论配一个企业应用案例 |
| 现状分析 | 企业真实数据+行业对比 |
| 问题诊断 | 理论工具如同"体检报告" |
| 方案设计 | 具体、可操作、有优先级 |
| 实施保障 | 组织、人才、资金、制度四维度 |
| 结论 | 管窥核心发现，不说空话 |

### 理论嵌入规范

引用理论库中的理论时，自动遵循以下格式：

```
[理论名称]（由[大师]提出）认为/指出/主张...
   → 定义（引用理论库中的定义）
   → 分析框架（引用理论库中的分析维度）
   → 案例应用（该理论在类似企业中的应用）
   → 本研究的适用性（为什么这个理论适合当前分析）
```

### 避免的写作陷阱

| 陷阱 | 示例 | 改进 |
|------|------|------|
| 空泛概念 | "提升管理水平" | "建立基于平衡计分卡的绩效管理体系" |
| 理论堆砌 | 一口气引用 5 个理论 | 选 1-2 个深度使用，其余辅助 |
| 有理论无案例 | "根据波特五力..." | "根据波特五力模型，A集团在供应商议价能力方面面临...（年报数据显示供应商集中度从2019年的...）" |
| 有分析无结论 | 列出数据但不说意义 | 每个数据后跟一句"这意味着..." |
