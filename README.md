## 概述

**MBA 写作舱** 是一个专门为 MBA（工商管理硕士）论文写作设计的 Claude Code Skill。它覆盖了从选题到成稿的完整写作流程，内置 194 条经过整理的管理理论条目、6 种 MBA 论文类型模板、以及 8 个阶段 12 个专业写作代理。

与通用写作工具不同，MBA 写作舱的核心差异在于：
- **理论驱动**：自动从 194 条理论库中为你的章节匹配合适的管理理论
- **大师视角**：内置德鲁克、波特、克里斯坦森等 8 位管理大师的理论视角
- **MBA 特化**：支持 MBA 学位论文、案例分析、商业报告、战略分析等 6 种论文类型
- **实战导向**：强调"理论→数据→实践"三元论证，符合 MBA 论文评审标准

## 核心能力

### 8 阶段写作流水线

```
Phase 0 ─── MBA Intake Agent         配置面谈：论文类型/学校/格式
Phase 1 ─── MBA Literature Strategist 商业文献 + 理论库检索
Phase 2 ─── MBA Structure Architect   7章论文/案例分析/商业报告
Phase 3 ─── MBA Argument Builder      理论-数据-实践三元论证
Phase 4 ─── MBA Draft Writer          实战风格/案例驱动
Phase 5ab ─ MBA Citation + Abstract   APA7+GB7714/商务双语
Phase 6 ─── MBA Peer Reviewer         MBA 论文五维评分
Phase 7 ─── MBA Formatter             学校模板/答辩PPT
```

#### Phase 0 — 配置面谈（Intake）

启动写作前的必要环节。系统会引导你完成 10 项配置：

1. 研究主题与核心问题
2. 论文类型选择（MBA 学位论文 / 案例分析 / 商业报告 / 战略分析 / 企业诊断 / 行业研究）
3. 目标院校（用于匹配格式要求）
4. 学科领域
5. 引用格式（默认 APA 7 + GB/T 7714）
6. 输出格式（Markdown / DOCX / PDF / PPT）
7. 语言（中文 / 英文 / 中英双语）
8. 建议字数（MBA 论文通常 20000-50000 字）
9. 已有材料
10. 理论偏好

#### Phase 1 — 文献策略（Literature）

自动检索商业文献来源，包括：
- **学术期刊**：HBR、管理世界、南开管理评论等
- **咨询报告**：McKinsey、BCG、Deloitte 等
- **行业报告**：艾瑞咨询、易观分析、36氪研究院等
- **企业数据**：上市公司年报、招股说明书

同时扫描理论库，将相关理论的关键著作纳入参考文献搜索范围。

#### Phase 2 — 结构设计（Structure）

根据论文类型自动选择结构模板：

| 论文类型 | 结构 | 建议字数 |
|---------|------|---------|
| MBA 学位论文 | 7 章结构 | 25000-50000 |
| 案例分析论文 | 6 段式 | 15000-25000 |
| 商业报告 | 5 段式 | 10000-20000 |
| 战略分析报告 | 6 段式 | 15000-30000 |
| 企业诊断报告 | 5 段式 | 10000-20000 |
| 行业研究报告 | 6 段式 | 15000-30000 |

#### Phase 3 — 论证构建（Argument）

采用 MBA 特有的"理论-数据-实践"三元论证模式：

```
理论透镜 → 企业数据/案例 → 实践建议
```

自动预判 4 类常见反对意见并准备回应策略。

#### Phase 4 — 全文撰写（Draft）

遵循 5 条 MBA 写作核心原则：
1. **实战导向** — 每个理论分析连接企业实践
2. **案例驱动** — 用具体案例支撑抽象理论
3. **问题解决** — 以解决管理问题为核心目标
4. **数据支撑** — 避免纯定性分析
5. **可操作性** — 分析结论能转化为管理行动

#### Phase 5 — 引用与摘要（Citation + Abstract）

- 双格式引用：APA 7 + GB/T 7714
- 企业数据专用引用格式（年报、行业报告、百科）
- 商务术语中英对照表
- MBA 关键词库（按研究方向分类）

#### Phase 6 — 同行评审（Reviewer）

模拟 MBA 论文评审委员会三方视角，五维度评分：

| 维度 | 权重 | 关注点 |
|------|------|--------|
| 实践价值 | 30% | 方案是否可落地 |
| 理论应用 | 25% | 理论使用是否准确深入 |
| 分析深度 | 20% | 数据分析和案例剖析 |
| 逻辑结构 | 15% | 论证是否严密 |
| 写作规范 | 10% | 格式和语言规范 |

三方评审视角：论文导师（学术规范）、企业导师（实践价值）、学术评审（理论创新）

#### Phase 7 — 格式输出（Formatter）

- 学校论文格式（页边距、字体、行距、封面）
- 答辩 PPT 自动生成（11 页标准结构 + 演讲备注）
- 输出格式：Markdown / DOCX / PDF / PPTX

### 6 大特色模块

#### ① 理论推荐引擎（Theory Recommender）

**触发词**：`推荐理论` / `用什么理论`

根据你描述的写作场景，自动从 194 条理论库中匹配 Top 3-5 个管理理论，每个包含适用理由、分析框架、关键文献和注意事项。

**示例**：
> "我在写第三章，分析某集团的 APP 用户运营现状"
>
> → 自动提取关键词：用户运营、APP运营、现状分析
> → 推荐：服务利润链、价值共创理论、服务蓝图、用户生命周期管理

#### ② 段落逻辑检查（Logic Checker）

**触发词**：`检查逻辑` / `帮我审一段`

六维度逻辑检查：
1. 现象描述完整性（5W1H）
2. 理论支撑（是否匹配、是否准确）
3. 论证链条（现象→理论→分析→结论 是否完整）
4. 结论与前文呼应
5. 学术语言规范
6. 引用规范

#### ③ 分析模板生成（Template Generator）

**触发词**：`生成分析框架` / `用XX分析XX`

为指定理论生成结构化的分析表格，包含维度、分析要点、数据来源建议和填写提示。预置模板包括：波特五力、SWOT、价值链、PEST、VRIO 等。

#### ④ 大师视角匹配（Master Match）

**触发词**：`大师怎么看` / `视角匹配`

将管理问题抽象化后，匹配 2-4 位管理大师的理论视角，提供分析框架、可能的创新点和理论局限。支持"大师辩论"模式——让不同大师视角交锋。

**内置大师库**：
- 彼得·德鲁克 — 目标管理、事业理论
- 迈克尔·波特 — 五力模型、通用战略、价值链
- 克莱顿·克里斯坦森 — 颠覆性创新、JTBD
- 亨利·明茨伯格 — 战略历程、组织构型
- 野中郁次郎 — SECI 模型、知识创造
- 吉姆·柯林斯 — 基业长青、飞轮效应
- 普拉哈拉德 — 核心能力、价值共创
- 卡普兰&诺顿 — 平衡计分卡、战略地图

#### ⑤ 查重预警（Plagiarism Checker）

**触发词**：`查重` / `检查抄袭`

将用户内容与理论库原文进行相似度比对，分三级预警（高>60% / 中 30-60% / 低<30%），对高度相似段落提供 3 种改写方案（学术改写、案例结合、问题导向）并附正确引用格式。

#### ⑥ 引用格式生成（Citation Generator）

**触发词**：`生成引用` / `引用格式`

支持 APA 7 + GB/T 7714 双格式自动生成，可批量处理。输入"引用德鲁克 目标管理"即可获得两种格式的标准引用。

## 使用方式

### 全流水线模式

适合从头开始写一篇完整的 MBA 论文：

```
帮我写一篇MBA论文，题目是XX公司的XX问题研究
```

```
帮我规划MBA论文——我想研究新能源车企的用户运营
```

系统会从 Phase 0 配置面谈开始，逐步引导你完成全部 8 个阶段。在每个阶段结束时，你可以审阅产出物并决定是否回溯修改。

### 单模块模式

适合已有草稿、需要专项支持的场景：

| 你的需求 | 怎么说 |
|---------|--------|
| 需要理论支撑 | `推荐理论——我在写第三章用户运营现状分析` |
| 检查草稿质量 | `检查逻辑——[粘贴草稿]` |
| 生成分析框架 | `用波特五力分析华为手机业务` |
| 获取大师视角 | `大师怎么看——员工流失率居高不下` |
| 检查是否抄袭 | `查重——[粘贴内容]` |
| 生成标准引用 | `生成引用——德鲁克 目标管理 1954` |
| 修改论文 | `修改MBA论文——[粘贴需要修改的部分]` |
| 转换格式 | `转PDF` / `转学校格式` |
| 写摘要 | `写MBA摘要——[粘贴论文全文]` |

### 组合使用

MBA 写作舱可以与其他 Skill 组合使用：

```
academic-paper + mba-writing-pod
→ MBA 论文全流程写作 + 理论智能匹配

deep-research + mba-writing-pod
→ 深度研究 → MBA 论文自动写作

mba-writing-pod + academic-paper-reviewer
→ MBA 论文同行评审（外部多角色审稿）
```

## 典型使用场景

### 场景 1：从零开始写毕业论文

> **用户**：我是 MBA 学生，想写关于传统企业数字化转型的论文，但不知道从什么角度切入。
>
> **使用方式**：
> 1. 输入 `帮我规划MBA论文——传统制造企业的数字化转型`
> 2. 系统启动 Phase 0 配置面谈，确定论文类型、学校格式
> 3. Phase 1 自动检索数字化相关的商业文献和理论
> 4. Phase 2 生成 7 章论文结构
> 5. 在 Phase 3 获得"颠覆性创新 + SECI 模型"的论证框架
> 6. Phase 4 逐章撰写，每章可单独调用逻辑检查
> 7. Phase 5-7 完成引用、摘要、评审和格式输出

### 场景 2：论文写到一半卡住了

> **用户**：我写完了现状分析，但不知道用什么理论来做问题诊断。
>
> **使用方式**：
> 1. 输入 `推荐理论——我在写第四章问题诊断，研究对象是某汽车集团的APP用户运营`
> 2. 系统匹配服务利润链、价值共创、用户生命周期管理等理论
> 3. 选择后输入 `生成分析框架——用服务利润链分析某汽车集团APP`
> 4. 获得结构化分析表格，按维度填写企业数据

### 场景 3：修改润色

> **用户**：初稿写完了，但感觉论证不够有力，有些段落可能和百科太像。
>
> **使用方式**：
> 1. 输入 `检查逻辑——[粘贴段落]` → 获取逻辑诊断报告
> 2. 输入 `查重——[粘贴段落]` → 获取相似度报告和改写建议
> 3. 输入 `大师怎么看——[描述核心管理问题]` → 获得多视角深化建议
> 4. 输入 `修改MBA论文——[粘贴全文]` → 系统化修改

### 场景 4：准备答辩

> **用户**：论文写完了，需要做答辩 PPT。
>
> **使用方式**：
> 1. 输入 `转学校格式` → Phase 7 自动生成 11 页答辩 PPT 大纲
> 2. 每页含演讲备注和设计建议
> 3. 输出 PPT Markdown，可用 Marp/Pandoc 转换为 PPTX

## 注意事项

### 使用建议

1. **先规划后写作**：建议从 `plan` 模式开始，确定选题和理论框架后再进入全流水线
2. **逐章检查**：每写完一章，使用 `检查逻辑` 模块验证论证完整性
3. **理论深度优于广度**：选择 1-2 个核心理论深度使用，而非堆砌多个理论
4. **数据支撑**：MBA 论文评审重视数据，尽量使用企业真实数据而非纯定性描述
5. **及早查重**：写完初稿后使用查重模块检查，避免后期大幅修改

### 已知限制

- 学校格式模板目前覆盖常见院校格式，部分学校可能需要手动调整
- 理论库为 194 条通用管理理论，不包含特定行业深度理论
- 本 Skill 聚焦论文前中期（选题→初稿），评审和答辩环节的支持相对精简
- 生成内容为 AI 辅助产出，最终论文质量取决于用户的审阅和修改

### 隐私与安全

- 论文内容在本地处理，不会上传至外部服务
- 理论库为公开管理知识的整理，不含任何个人信息
- 所有模板中的姓名、导师、学校等字段均为占位符，需用户自行填写
