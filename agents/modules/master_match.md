# Master Match — 大师视角匹配

## 触发条件

用户输入包含以下关键词时激活：
- "大师怎么看" / "大师视角"
- "视角匹配" / "换一个角度"
- 用户描述一个管理问题并问"从什么理论视角切入"

## 工作流程

### Step 1: 问题抽象化

用户描述管理问题 → 抽象为理论问题的类型：
```
"A集团员工流失率持续升高"
→ 这是一个"员工激励与保留"问题
→ 核心变量: 员工满意度、薪酬公平性、职业发展

"公司数字化转型项目总是失败"
→ 这是一个"组织变革管理"问题
→ 核心变量: 变革阻力、能力建设、领导力
```

### Step 2: 大师匹配

为每个抽象后的问题类型，匹配 2-4 位大师的理论视角：

```markdown
## 问题: [用户问题]

### [视角1] [大师] — [理论名称]

**如果由这位大师来研究...**
> [研究问题建议]

**为什么匹配:**
[因为该理论的核心关注点与用户问题一致]

**分析框架:**
[该理论的分析维度]

**可能的创新点:**
[在经典理论上做情境化延伸的方向]

**局限:**
[该理论解释范围之外的因素]

---

### [视角2] [大师] — [理论名称]
...
```

### Step 3: 辩论模式（可选）

用户选择 2 个视角后，可触发"大师辩论"：
- 视角 A 会如何质疑视角 B 的分析
- 视角 B 如何回应
- 综合后哪个视角更适合本研究
