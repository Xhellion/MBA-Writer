# MBA 写作风格指南

## 核心原则

1. **问题导向** — 每一章都要回答一个管理问题
2. **案例驱动** — 用具体案例支撑抽象理论
3. **数据支撑** — 避免纯定性分析
4. **可操作** — 分析结论转化为管理建议

## 章节写作规范

### 绪论
- 从行业痛点切入，用数据说话
- 明确研究问题和研究边界
- 典型错误：背景写得太宏大，与论文主题脱节

### 理论综述
- 每个理论配 1-2 个企业案例
- 理论之间要有逻辑关联，而非简单罗列
- 典型错误：从知网复制粘贴综述，未做归纳总结

### 现状分析
- 企业真实数据 + 行业对比
- 使用图表展示趋势
- 典型错误：只有文字描述，缺少数据支撑

### 问题诊断
- 基于理论工具的"体检报告"
- 根因分析（5 Whys + 理论验证）
- 典型错误：浅尝辄止，只看到表面问题

### 方案设计
- 至少 2 个方案的比较
- 推荐方案需有评分依据
- 典型错误：方案太理想化，不接地气

### 实施保障
- 组织、人才、资金、制度四维度具体化
- 典型错误：只说"加强管理"、"提高重视"

### 结论
- 核心发现 3-5 条
- 不说空话套话

## 避免的写法

| 避免 | 原因 | 改进 |
|------|------|------|
| "随着市场经济的发展..." | 烂俗开头 | 直接说行业痛点或数据 |
| "加强管理"、"提升水平" | 空泛 | 具体到"建立XX制度"、"引入XX工具" |
| 理论堆砌（2页8个理论） | 变相凑字数 | 精选2-3个深度使用 |
| 有分析无结论 | 数据白列 | 每个数据后跟"这意味着..." |
| 抄袭百科原文 | 查重必杀 | 用自己的话+正确引用 |
| "笔者"、"本文作者" | 非学术 | 直接用"本研究"、"本文" |

## 学术语言 vs 口语化

| 口语化 | 学术化 |
|--------|--------|
| 很多企业 | 多数企业、70%以上的受访企业 |
| 越来越重要 | 重要性日益凸显 |
| 大家觉得 | 受访者普遍认为 |
| 东西 | 产品、服务、要素 |
| 搞一个方案 | 设计一套解决方案 |
| 越来越差 | 呈现持续下降趋势 |

## 理论引用规范（防止查重）

1. **不要直接复制百科定义** → 用自己的话重述 + 引用来源
2. **理论+案例绑定引用** → 引用理论定义后，立即配企业案例
3. **问题导向引入** → 不是"根据XX理论..."，而是"为什么出现这个问题？XX理论提供了一个分析视角：..."
